## 4. Priorities Roadmaps

### Short-Term (0–3 Months)

- Publish and socialize the Agent Mindset Guide.
- Train SE teams on AI-first workflows.
- Launch the first version of agents using lower autonomy.
- Conduct bi-weekly retrospectives to improve agent performance and workflows.
- Define governance and quality standards.

### Mid-Term (3–6 Months)

- Deploy agents to pilot projects and track adoption and performance metrics.
- Implement a standardized intake checklist and governance review process.
- Expand the prompt and skills Marketplace with reusable assets.
- Roll out structured enablement sessions across Solution Engineering.

### Long-Term (6–12 Months)

- Scale agent adoption across multiple SE projects and regions.
- Increase agent autonomy based on performance, reliability, and governance readiness.
- Integrate agents into SE delivery workflows and operational processes.
- Establish dashboards to monitor adoption, productivity, and quality metrics.
- Continuously refine standards, playbooks, and best practices based on real-world usage.

## 5. Proposed Metrics and Goals

### Automation Targets by Roadmap Phase

[SWE Bench](https://www.swebench.com/)

| Phase                   | Target Automation Level | Description                                                  |
| :---------------------- | :---------------------- | :----------------------------------------------------------- |
| Short-Term (0–3 Months) | 30%                     | Automate repetitive and well-defined tasks with human oversight and validation. |
| Mid-Term (3–6 Months)   | 40%                     | Expand automation across pilot projects with standardized workflows and governance. |
| Long-Term (6–12 Months) | 60%                     | Scale reliable agent-driven workflows with increased autonomy and reduced manual intervention. |

### Adoption Metrics

- Number of agents/skills created and deployed.
- Percentage of Solution Engineers using AI tools weekly.
- Training completion rates.
- Number of AI-enabled projects.

### Productivity Metrics

- Reduction in delivery time.
- Increase in automation of repetitive tasks.
- Time saved per project.
- Improvement in proposal and implementation turnaround times.

### Quality Metrics

- Accuracy and reliability of AI-generated outputs.
- Reduction in defects and rework.
- Compliance with AI quality standards.
- Agent performance and success rates.

### Enablement Metrics

- Number of workshops and training sessions delivered.
- Number of playbooks and reusable assets published.
- Documentation adoption and usage metrics.
- Team satisfaction and engagement scores.
